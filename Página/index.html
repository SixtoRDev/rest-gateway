API Gateway Empresarial

CONECTA es un API Gateway centralizado, diseñado para ser el único punto de entrada y salida para todo el tráfico de APIs de la organización. Actúa como una capa estratégica de gestión que introduce gobernanza, seguridad, control y observabilidad sobre los flujos de comunicación.

Agilidad Operativa

Reduce la dependencia de desarrollos específicos para conexiones con otros sistemas, acelerando el intercambio de información.

Mejora en el desarrollo

Otorga autonomía a los equipos de desarrollo para exponer y consumir APIs de forma rápida y segura a través de una UI de administración.

Centralización y Consistencia

Asegura que todas las políticas de seguridad, auditoría y enrutamiento se apliquen de manera uniforme desde un único punto de control.

Flujos de Datos Clave

Visualiza los flujos de interacción más críticos del sistema paso a paso. Utiliza los botones para cambiar entre los diferentes flujos.

Flujo de Procesamiento de Petición Entrante Válida

  1. Petición HTTPS llega al Gateway Core.
  2. Filtro de Auditoría intercepta y envuelve la petición.
  3. Filtro de Autenticación valida el token JWT.
  4. Motor de Enrutamiento reenvía petición al servicio interno.
  5. Servicio Interno devuelve la respuesta.
  6. Filtro de Auditoría intercepta respuesta, ensambla la traza y la envía.
  7. Gateway Core devuelve la respuesta final al cliente.

Flujo de Actualización de Ruta

  1. Admin guarda cambios de ruta en la Admin UI, que llama al Config Service.
  2. Config Service actualiza la base de datos de configuración.
  3. Config Service publica un evento `ConfigChangedEvent` en un Message Broker.
  4. Instancias del Gateway Core reciben el evento.
  5. Gateway Core dispara un refresco interno, recargando las rutas desde el Config Service (y su DB) sin reinicio.

Modelo de Dominio

El diseño del dominio define las entidades clave del sistema. Haz clic en cada objeto para ver sus atributos.

ApiRoute

Representación de una regla de enrutamiento, análoga a `RouteDefinition` de Spring Cloud Gateway.

  • - String id
  • - String uri
  • - int order
  • - Map<String, Object> metadata
  • - List<PredicateDefinition> predicates
  • - List<FilterDefinition> filters

AuditTrace

Modela una entrada de auditoría completa para una única transacción a través del gateway.

  • - UUID traceId
  • - String routeId
  • - String requestMethod, requestUri, clientIp
  • - int responseStatusCode
  • - long processingTimeMs
  • - String requestHeaders, requestBody
  • - String responseHeaders, responseBody
  • - ZonedDateTime timestamp

PredicateDefinition

Representa una condición de coincidencia para activar una ruta (ej. por Path, Method, Header).

  • - String name
  • - Map<String, String> args

FilterDefinition

Representa una acción a realizar sobre la petición o la respuesta (ej. modificar path, añadir headers).

  • - String name
  • - Map<String, String> args
